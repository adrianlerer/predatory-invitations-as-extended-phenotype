# Extensions Directory

This directory explores applications of the Parasitic Spontaneous Order (PSO) framework beyond predatory academic journals.

## Concept

If PSO is a general pattern of institutional parasitism, it should appear in other domains where:

1. Symbolic credentials have value
2. Verification costs are high
3. Mimicry of legitimate signals is cheap
4. Incentive structures reward quantity over quality

## Proposed Extensions

### 1. Citation Rings (`citation_rings.md`)

**Phenomenon:** Networks of papers that cite each other to inflate metrics

**PSO Markers:**
- Mimics legitimate scholarly discourse (references, citations)
- Extracts resources (journal impact factors, researcher h-indices)
- Functional divergence (cites don't indicate quality, only network membership)
- Emergent order (no central coordination, self-organizing)

**Research questions:**
- Do citation rings exhibit spontaneous order properties?
- Can EPT explain their stability?
- Are they evolutionarily stable strategies?

---

### 2. Conference Mills (`conference_mills.md`)

**Phenomenon:** Low-quality academic conferences that charge high registration fees

**PSO Markers:**
- Mimics legitimate academic conferences (proceedings, presentation slots)
- Extracts resources (registration fees, hotel commissions)
- Functional divergence (no rigorous review, acceptance near 100%)
- Minimizes friction (online presentations, minimal engagement)

**Parallel to predatory journals:**
- Both exploit "presented at conference" credential value
- Both use scraping to invite participants
- Both have rapid acceptance timelines

---

### 3. Ranking Manipulation (`ranking_manipulation.md`)

**Phenomenon:** Institutions gaming university/program rankings through reporting manipulation

**PSO Markers:**
- Mimics legitimate quality signals (selectivity, research output)
- Extracts resources (student tuition, research funding)
- Functional divergence (reported metrics ≠ actual quality)
- Stable because verification is costly

**Connection to PSO:**
- Rankings create selection pressure
- Mimicry of quality signals is cheaper than actual quality improvement
- System-level adaptation emerges without central planning

## Cross-Domain Pattern Recognition

### Common Elements Across PSO Instances

| Feature | Academic Journals | Citation Rings | Conference Mills | Ranking Games |
|---------|-------------------|----------------|------------------|---------------|
| **Credential valued** | "Published paper" | "Highly cited" | "Conference presentation" | "Top-ranked institution" |
| **Verification cost** | High (check venue quality) | High (analyze network) | High (assess rigor) | High (audit reports) |
| **Mimicry cost** | Low (ISSN, DOI cheap) | Low (reciprocal cites) | Low (website, proceedings) | Moderate (reporting) |
| **Functional divergence** | Review ≠ validation | Cites ≠ quality | Conference ≠ peer review | Rank ≠ quality |

**Pattern:** Same fundamental dynamics across different domains

## Theoretical Predictions

### If PSO is General Framework

**We should observe:**

1. **Emergence without coordination:** PSO instances arise independently
2. **Mimetic convergence:** Different PSO types use similar legitimacy signals
3. **Niche specialization:** PSO diversifies to exploit different credential types
4. **Persistence despite exposure:** PSO stable even when known
5. **Evolutionary arms race:** Mimicry sophistication increases as detection improves

**Empirical test:** Do these predictions hold across domains?

## Practical Applications

### For Policymakers

**If PSO is general pattern, then:**

- **Sectoral solutions insufficient:** Fixing journals won't eliminate conference mills
- **Systemic approach needed:** Address underlying incentive structures
- **Anticipate emergence:** New PSO types will appear in credential-dependent systems

### For Institutional Design

**Principles:**

1. **Reduce credential dependence:** Diversify evaluation beyond simple metrics
2. **Increase verification:** Make quality checking less costly
3. **Align incentives:** Reward actual contributions, not token accumulation
4. **Transparency defaults:** Require disclosure of processes, not just outcomes
